# /cw-handoff — 结构化交接（防信息丢失）

把当前对话蒸馏成一份**结构化交接文档**，让一个**全新、零上下文的 agent** 能接着干、不用回头问用户重讲。
输入 `$ARGUMENTS`：若有 = 下一 session 的聚焦点，按它裁剪文档。

## 核心原则（像 /cw-clarify 结构化需求那样结构化交接）
- **抓"只在对话里、会随对话消失"的东西**：决策与理由、坑/死路、当前状态、恢复路径——**这些不在任何制品里**，丢了就重走弯路。
- **已落盘的不复述**：PRD/计划/ADR/commit/diff/分析文件 → **按路径引用**，不抄内容。
- **证据分级 + 诚实**：标 ■已验证 / ○未验证/假设；**如实写阻塞、失败、撞过的死路**（不只报喜）——下个 agent 最需要知道哪里是雷。
- **质量门（自检）**：一个没参与过对话的 agent，只读这份文档 + 引用的制品，能不能不问用户就接着干？不能 → 缺了关键上下文，补。
- **注意力预算**：承重信息求全，废话求无；空的小节直接删，不硬凑。

## 输出结构（写进文件；标 ★ 为必填，其余按需）
```markdown
# Handoff: <一句话任务名> · <日期>
（若有聚焦参数：本次交接聚焦 <参数>）

## ★1 目标（含**最新/中途修正的**要求）
用户到底要什么（原话/原意优先，别用我的转述覆盖）；**特别标注 session 中途用户纠正过/改变过的要求**——这是新 agent 最易丢、丢了就走回头路的隐性上下文。

## ★2 现状
一句话定位：done / 进行中（卡在哪）/ blocked。完成度 + 此刻正在做的那一步。

## 3 冻结决策与理由（最高价值 — 防下个 agent 重新纠结或推翻已定的事）
- 决策：<定了什么>  理由：<为什么这么定>  代价/翻转触发：<何时该重估>
- （含被否决的方案 + 为什么否决——省得重试死路）

## ★4 下一步（有序、具体、可执行）
1. <下一个该做的动作，精确到文件/命令/接口>
2. …

## 5 活跃制品（引用不复制）
- <文件/PR/分支/issue 路径或 URL> — <一句话这是什么、改到哪>

## 6 坑 / 不要做（撞过的死路、约束、gotcha、反模式 + **本 session 犯过并纠正的错误类**）
- <雷点 + 为什么>
- **本 session 犯过并已纠正的（含方法纠偏，传给下个 agent 别重犯）**：<如"曾把 X 做成 Y，已改 Z，因为…">

## ★7 验证状态
- ■已验证：<什么，用什么证据（运行/测试/截图/你的-UX-探针工具）>
- ○未验证/假设：<什么还没证实——下个 agent 别当真>
- "怎样算完成(done)"：<验收标准>

## 8 待用户定（open questions）
- <需 Human 决策/澄清的事，别替他拍板>

## 9 恢复环境（怎么接着跑）
分支 / server 起法 / 关键命令 / 工作目录 / 数据状态。

## 10 建议下一 session 用的 skills/命令
- <如 /cw-change、/cw-experience-verify、/i-bug …>
```

## 落盘
1. **默认落 active run 的 run-dir `<run_root>/<thread-id>/`**（**不是 run_root 根**，避免写共享根致多会话争用；无 active tg run 才 `mktemp -t handoff-XXXXXX.md` 新空文件直接写；写已存在的项目约定位置如 `.claude-insights.md` 则先 Read 再 Write）。**引用的制品标"■路径已确认 / ○未验证"，跨仓或路径可能不可读时把最小承重摘要内联**（别让下个 agent 拿到失效指针）。
2. 返回：文件路径 + 一句话摘要（caveman 友好）。

> 与 §A 关系：handoff = "File-is-State"（对话会消失文件不会）+ "注意力预算"（蒸馏摘要）+ "诚实"（如实标验证状态与阻塞）的落地。质量来自**结构**——无结构的"随便写个交接"必丢决策理由与坑（这正是 /i-handoff 的缺陷）。
