# TG-THINK — 深碰撞·和用户共创把"想不清"钻成"想清楚"（谱系最深档）

> 环节：`/cw-think <scope>`。解**认知模糊**——连该建什么/该验什么、甚至"真问题是什么"都不清楚，核心信念可能是错的。
> 不建活规格/不写码（那是 spec/实现），但**深碰结论落盘 `docs/topics/`**（`tbat`/`tcopy`/`tinit` = i-think 生态的取回/复制/初始化命令，依赖此固定位置；跨 session 复用）。
> 加载链按命令 `cw-think.md`（本卡 + `WF-TG-GUARDRAILS.md` 共享护栏 + `WF-TG-00-ORCHESTRATOR.md` 路由）。**深碰方法本身自包含**——深度来自下面每个动作真做到、**不靠读 workflow_ng 等外部方法论**（GUARDRAILS/ORCHESTRATOR 是共享护栏/路由，不是深度来源）。

## 何时用
**纯深碰**：核心信念可能错 / 真问题没锁定 / 极复杂新域 / brainstorm 选完发现底层信念存疑。低频重型。
- 只"不知选哪条路" → `/cw-brainstorm`（别杀鸡用牛刀）；只"没说清" → `/cw-clarify`；需求已清晰 → 直接 `/cw-spec`/`/cw-change`。

## 输入
`<scope>` + Human 的模糊想法（可能连方向都没有，只有一个困惑）。

## 流程（和用户**共创**，不是 AI 独白：每轮只推一个尖点带你的立场，让用户 react）

### 1. 先锚定真问题（别瞎钻）— 钻之前先搞清在解什么
- **先验证题面预设是否真成立（防伪命题，第一步）**：题面假定的对象/约束/现状，先 grounding（查代码/数据/记忆）确认它**真存在、真还成立**——很多"深度问题"其实是伪命题（对象已不存在 / 约束早被改了 / 记忆有误）。一旦证伪 → 直接上报"这是伪命题：X 其实是 Y"，**别去深碰一个不存在的东西**。
- **意图链回溯**：从用户当下诉求往前追"为了什么、再为了什么"，找到那个**切面**——既不被某个工具/实现绑死（太具体），又是能完整交付的（不太抽象）。
- **切面四问**（全过才往下，否则沿意图链继续调）：① 换个工具/做法，问题还在吗？(在=独立于实现) ② 只靠我们能完整解决吗？ ③ 单独解决、用户认这价值吗？ ④ 有别人难复制的优势吗？
- **共创**：把回溯出的切面 + 你的推荐一句话亮给用户确认——**别自己闷头定真问题**（一次一尖点的共创没这步、会瞎钻；这步是关键补强）。

### 2. 问题空间（和"解空间"分开——AI 最容易在这塌）
- **AI 天生想跳到"给方案"（解空间是舒适区）→ 先忍住。** 先把问题想清楚：**谁 → 为了什么进展 → 被什么挡 → 不解决会怎样**（白话，禁带实现词汇/功能列表）。
- 先允许乱讲（故事/反例/"绝对不能发生的事"），再挤干成结构——**过早结构化会裁掉"不好归类但真实存在"的问题**。
- **冲突/张力是一等公民**：问题集里没张力 = 没想够深，继续。
- **穷举后统摄成一句话**——"根本冲突到底是什么"。说不出 = 还在问题空间，**别进解空间**。

### 3. 碰撞钻探（到第一面墙 + 信念真的变没变）— 深度在这
- 沿因果链/"为什么"追到**第一面墙**：追不动、只能靠证据或假设撑住的底层点。
- **阻力层（碰到卡点逐个问）**：① 我在验证什么？ ② 暴露了什么领域硬事实？ ③ 这墙能穿透、还是基岩(穿不过)？ ④ **我的信念从 X 变成 Y 了吗？**——答不出(信念没变)→ 写 "**无修正：继续钻 / 证据债 / 显式 defer**" 三选一，别跳过。 ⑤ 这改变了上层哪个判断（回去改）？
- **≥2 条对立路径，且撞到的墙不同才是真候选**（墙相同 = 伪候选 = 换个说法）；重要议题至少追 3 层，或说明为何停。
- **好发现长这样**（区分真发现 vs 普通分析）：循环(A→B→C→A) · 反直觉反转(更多 X → 更少 Y) · 深层耦合(两个看似独立的机制共享一个瓶颈) · 反常识(推翻一个默认假设、且改变了某个决策)。**结论若能从题面直接推出 = 还在表面，继续钻。**
- **实干校验 + 证据门控（别在假设上建楼）**：下结论前必列三行——**已查证**(指出证据) / **待查证**(这就去查代码/数据) / **未验证风险**(标 ○ + 什么证据会推翻它)。全是假设没查证 = 不准下结论；答不上就向用户要真相（客户数据/运营数字/竞品）。
- **反事实挑战**（结论依赖某个主导解法时必做）：拿掉它，问题定义还成立吗？不成立 = 它是真轴心，动它要回退重想。

### 4. 三域覆盖（按改动选重心，不必全跑）
- **价值域**：对谁的什么结果负责 · 什么场景 · 承诺到什么程度 · 明确不做什么。
- **兑现域**：靠什么逻辑/方法让结果真发生 · 缺哪个能力就不闭环 · 什么证据算"真做到"。
- **实现域**：用什么路径做成可运行 · 跑起来真实会怎样 · 失败/恢复 · 什么取舍/技术债。
- **每个重心域钻一条微树**（防"报菜名"）：价值=触发情境→想完成什么→卡在哪；兑现=承诺靠哪条链→哪环会断→缺什么能力；实现=怎么跑→怎么失败→怎么恢复。
- 一个域碰墙可能改另一个域的前提 → 回去改（跨域回写是正常设计运动）。

## 贯穿全程：AI-native 自检（防 LLM 习惯性塌方 — 这是"激发 AI 思维"的真引擎）
- **别谄媚**：用户的、我自己的想法都要真撞。`BAD：这思路有意思可以深入` → `GOOD：这思路第一面墙在 X，我判断会败因为 Y；除非 Z 成立我才改立场`。
- **别假碰撞**：连续几轮没推翻过自己 = 确认偏差，回去找反例。
- **别假设洗白**：没验证的标"假设 ○"，别偷偷当事实用。
- **不确定就说**："我这里没把握"比流畅地编更有价值。
- **协议服务思考**：上面任何格式若挡住真实认知线索 → 扔掉它，回到问题本身。

## 共创节奏（一次一尖点 · 让用户 react）
- **一次只推一个尖点 + 你的立场/推荐**，让用户停下 react（同意/改）；别一次抛一堆（让人懵），别 AI 独白。
- **陌生域 offer 盲区扫描（非强制，命中才提）**：Human 明显在陌生域（不知道该问什么/不知道"好"长什么样）→ 先把该域关键维度/常见坑/"好"的样子教给 Human，让其有能力 react，再回深碰——别对着无法评判的人独自钻。
- **能从代码/现状/外部研究查到的，自己查，别问用户**（外部 scouting 如选型/资料标 ○ 直到实测；重型量化对比交 `/cw-experiment`）。
- 按依赖排序：先碰"别的判断依赖它"的那个。

## 退出门 + 三态（防伪深的硬门控 — 过不了不准往下）
- **三态**：每个判断标 **锚定**(说清主张) → **校验**(经得起挑战) → **冻结**(撞墙做了取舍 + 代价/条件)。**别把"说清"当"冻结"。**
- **门 1**：真问题没和用户确认 → 不准进解空间/深钻。
- **门 2**：没有 ≥2 条"墙不同"的对立路径 → 不准冻结。
- **门 3**：本轮信念没 before→after（也没"继续钻/证据债/defer"的合法理由）→ 不准结晶。

## 落盘（复用 i-think 机制，`tbat`/`tcopy` 继续生效）
存 **`{项目}/docs/topics/`**（不是隐藏目录）。注意：`docs/topics` 是 **i-think 跨工程约定位置**（`tbat`/`tcopy`/`tinit` 依赖此固定位置）——**非 per-project CONFIG 解析**（它是共享生态固定位置，不是工程实例路径；项目另有约定才覆盖）：
1. 创建 **per-topic 目录** `docs/topics/<topic_id>-<slug>/`（这是真相、每条一目录并发安全；topic_id 沿用 i-think `TH-<日期>-<slug>` 可读可排序；**同日同 slug → 原子 `mkdir` 失败则自动追加 `-2`/`-3` 递增**，不赌不撞）；再注册到 `docs/topics/TOPIC-BOARD.md`（无则从 `~/.cliworkflow/workflow_tg/templates/TOPIC-BOARD.md` 建，状态 `DISCUSSING`）——**board 仅可再生索引，并发丢 entry 可从各 topic 目录重建，别当唯一真相**。
2. 写 `docs/topics/<topic_id>-<slug>/` 下 session 文件，**每轮 append 一个 Round**（Human 原话 + 本轮信念修正 before→after + 新发现/回写），**绝不覆盖旧认知**（File-First：对话会消失，文件不会）。**session 文件头记 `status`(DISCUSSING/MERGED)+结晶数 = 状态真相**（board 只是它的聚合索引）。
3. 收尾：把最终 `status`(如 MERGED)+结晶数写进 **session 文件头（状态真相）**，再同步 TOPIC-BOARD 索引——**board 可从各 topic 文件头聚合重建，并发丢 entry 不丢真相**。

## DONE-WHEN
- 真问题锚定（切面四问过）+ 核心假设被验证或修正（有 before→after）+ 每个待建场景说得出"用户做到什么算成"。
- 收敛判据 = **可验**，不是哲学完备。够验就停。

## 去向
- 场景清单逐条 → `/cw-spec <scope>`（新建活规格）。
- 发现其实只"选项没定" → 降 `/cw-brainstorm`；只"没说清" → 降 `/cw-clarify`。

## 纪律（防伪深四连）
- 信念全程没出现"原以为→现在" = 没真撞 → 回去钻。
- 全程只有一条路 = 冥想不是碰撞。
- 结论全 ○（无证据）= 没钻到底 → 去 grounding。
- 第一轮就"方案 A/方案 B" = 跳过了问题空间 → 退回问"到底在解什么"。
