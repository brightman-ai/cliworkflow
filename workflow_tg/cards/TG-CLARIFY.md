# TG-CLARIFY — 轻量富化（表达模糊）

> 环节：`/cw-clarify <scope> "<人话>"`。把 human-style 模糊需求富化为清晰可验诉求。解**表达模糊**（"心里有数没说清"）。
> 先读 `WF-TG-GUARDRAILS.md`。

## 何时用
需求大致清楚但表达模糊。比 /cw-think 轻（不深碰撞）。是统一前门式的"需求放大器"。

## 流程
1. **先抓真意图、别顺字面跑偏**（三档共同起点；AI 易 pattern-match 字面就填，先想"他真正要的是什么、背景约束是什么"）：从人话 + 现有代码/活规格上下文**自动填**：意图 / 范围与非目标 / 验收清单 / 视觉对标 / 隐含意图 / 约束。能推断的填上并标 `[假设]`；**核心意图读不准 = 最高优先级的那 ≤3 问之一**。
2. **仅把"会改变实现方向、且无法从上下文推断"的留白**，提成 **≤3 个**问题（优先级：验收标准 > 隐含意图/范围边界 > 非目标 > 视觉对标参照）。绝不一次问一堆、不问能自己查代码得到的。
3. 产出一份清晰可验诉求，**必须含这 7 要素（缺一不合格）**：① **意图** ② **范围锚定**（改哪块/在现有什么基础上/集成点；**同族实体矩阵 sweep**：同类对象/入口逐个标"覆盖"或"显式非目标"，防漏对称项）③ **显式非目标**（不做什么 + 为什么）④ **验收清单**（每条标 oracle、第一面墙可判定）⑤ **视觉对标项**（仅 UX：逐项列，不接受"按 X 范式"这种不可判定描述；参照物层级**源码 > 截图 > 文字描述**——有实现了目标行为/样式的代码、组件、库就直接指路径给它，信息量最大）⑥ **隐含意图显式化**（把"其实想要的 / 最可能被猜错处"写明）⑦ **验证可达性**（给本地可验的具体夹具/数据，并把**最难的冲突/边界情形显式列为验收项**，如"N 个测试账号的角色分布""支持覆盖默认值"这类可执行夹具）。自检：7 要素齐，且每条都可被验证。**落盘**：原话 + 富化诉求(7 要素) 存 `<run_root>/<thread-id>/request.md`（经 CONFIG 解析；下游 spec/change 消费、机器可解析、可 resume——GUARDRAILS §4 护栏4，矩阵³，别只留对话）。**快修分支例外**：按路由步的快修声明豁免 request.md，改为收尾 implementation-notes 一行回写。
4. **路由去向**：
   - 全新 portal/BS → 交 `/cw-spec <scope>`（相当于轻量 think → 建新活规格）。
   - 已有 portal/BS → 交 `/cw-change <scope>`（写 delta）。
   - **快修分支（G0 合法出口，替代静默跳过；仅限 A 类实现缺陷**——意图已明确、实现没达到；**意图变了哪怕一句话 → `/cw-change` 写 delta，不入快修**）：根因 ■ 已锤 + 改动小且可逆 → 可就地直修，三件强制：①开工回执一句"**本轮快修直落，不开 request.md**"；②**行为类改动（键盘/点击/时序/注入）ship 前必须 runtime 证据**（项目 CONFIG 声明的 UX 工具/harness）——确实不可验 → 显式写"未经独立验证，oracle=用户实测"+ 最可能失败面 + 回滚方式；③收尾 **一行记入 `implementation-notes.md`**（GUARDRAILS §8 契约、verify 对账消费：改了什么 + **部署状态**——已部署到哪个实例/未部署）。**排除项**：**扩大网络可达面**（tunnel/端口暴露/反代/分享链接）不入快修资格——先核目标服务门控现状，无门控即 G3 停。部署目标命中 memory/CONFIG 声明的**生产实例**（CONFIG 未声明环境 → 一律按生产对待）→ 先临时实例过 oracle，或获 Human 本轮显式豁免。

## DONE-WHEN
- 诉求清晰到"喂给实现/change 不用再回头问"；验收清单可被第一面墙判定。

## 纪律
- UX 类**必须**产出"逐项视觉对标"（不接受"按 X 范式"这种不可判定描述）。
- 需求意图 ≠ 实现字段：表达"为什么要"，提醒别照搬字段名。
- **单轮收敛、不支持多轮持续澄清**：一轮没澄清清 = 信号该**升档**——方向没定 → `/cw-brainstorm`；核心信念存疑 → `/cw-think`。不在 clarify 再问一轮（防撑胖、防与 brainstorm/think 重叠）。
- **同一 scope 短期内连续收到 UX 投诉（≥2 次，含本会话与可见记录）→ 停止逐点修**：主动 offer `/cw-experience-explore-full` 旅程级撞墙（逐点修 = 让用户当 QA；连环投诉本身就是 Witness 缺位信号）。
- **就地深碰的资产不丢**：在 clarify 内实际做了模型级设计（≥2 个耦合决策 / 动领域模型）→ 产物按 think 落盘规范进 `docs/topics/<topic_id>-<slug>/` 并登记 TOPIC-BOARD（哪怕没升 `/cw-think`）——跨 session 可取回；下次同类默认先升档。
- **No-human 降级**：有真人 → 照常 ≤3 问。**无真人在场（unattended/pipeline）→ 高杠杆问题用合理默认 + 标 `[假设]`，并列入产出的"待人确认"清单，不阻塞**；恢复有人时再确认。**降级上限**：允许继续富化/查证/落盘，**不授权开建实现**；结构性变更（UI 布局/信息架构/schema）unattended 只落方案不实施。证据级诚实：先验推断标 ○，别冒充 ■硬证据。

### 真实脏输入处理（实测补强 — 真实工程提示常见，纯文本合成测覆盖不到）
- **截图 = 读图（默认，不是降级）**：用户给截图**路径**（如 `tmp/clip/.../x.png`）或附图 → **用当前 runtime 的图片读取工具读它**（Claude Code=`Read`；其它 runtime 用其等价图片工具），从图里读出元素/状态/布局把视觉对标做实——**有实体文件就读，别标 ○、别要求人转述**。仅当 **runtime 无图片读取能力或确无可读图**（只说"如图"无路径/文件不存在）→ 标 ○ + 让 **Witness 在第一面墙截图读取**闭合（**机器读，非人读**），或回问"截图路径？"。
- **一条混多异质诉求 → 先拆再各自放大**：一条 prompt 混了不相干的多条线（如"改 toolbar" + "加通知引导"）→ **先拆成各自清晰诉求、分别标号路由**，不糊成一团；同源重复项合并、跨条引用去重。**`≤3 问`限针对每条独立诉求（非总数）**：拆后每条各自走"自动填→≤3问→产出 7 要素"。
- **反问/事实待证当独立输入类型**：诉求里的事实性反问（"pc 是不是 http 也行？""需要 https 吗？"）先**查代码/运行时求答**；若它决定实现分支且无法解析 → 标 `[待确认]` 上 Human，**不静默假设成事实**。
- **强指代不可解析**：脱离上一轮对话的"这个也导致…/同上" → 标"指代不明，需原始上下文/截图"，别猜一个对象往下做。
- **症状/回归类输入（"改出 bug / 又坏了 / 没生效"）→ 先钉三件再查根因**：①**持续 vs 瞬时**（一直坏，还是闪一下/偶发）②**同一实例吗**——先核**用户所测实例的构建时间 vs 改动提交时间**（版本代差 = 伪回归头号来源，结论一句话前置）③**最小复现或一个决定性观察**（能一次判掉各嫌疑的那种）。能稳定复现 → offer 转 `/cw-experience-troubleshoot`。
