# TG-BRAINSTORM — 方向逼问/收敛（D1：把模糊方向逼成拍定的方向 + 关键决策）

> 环节：`/cw-brainstorm <scope> "<想法>"`。解**"大方向有、不知选哪条路/没想全选项"**。介于 clarify(收敛) 与 think(深碰) 之间。
> 先读 `WF-TG-GUARDRAILS.md` + `WF-TG-00-ORCHESTRATOR.md`。

## 何时用
方向模糊、要把它**逼成可建的拍定方向**时（不只是看选项，是逐个拍定关键决策到共识）。**不是**"没说清"(→/cw-clarify)、**不是**"核心信念可能错/极复杂新域"(→/cw-think)。clarify 跑一轮发现"方向本身没定"→ 升这里。

## 输入
`<scope>` + Human 的模糊想法/方向。**批量已发现问题的反馈**（多截图/多条控诉）→ 先粗分诊（非正式对账，终判仍在 verify）：**实现缺陷**（意图明确没做到）列为修复项走 clarify 快修分支、**环境/工具问题**转 `/cw-experience-troubleshoot`（两者都不占逼问预算），其余（规格空白/意图变/品味）才进决策树。

## 流程（核心 = 自适应逼问式访谈，逼到"方向 + 关键决策都拍定"为止——不是丢一堆选项让你挑）
0. **先对齐意图/背景（不跑偏 — clarify/brainstorm/think 三档共同起点）**：动手逼问前，先用一两句复述"你真正想解决的是什么、背景/约束是什么"，让用户确认或纠正——**别顺着字面就把决策树走偏**（AI 易 pattern-match 一个貌似合理的解释直接跑）。复述**用用户自己的词汇**（新术语带一行白话定义），并点名"**主场景/主要调用者是谁**"；依赖在途侦察的判断，等证据回来再抛（可先只对齐意图）。背景能从代码/现状查到的先自己查。
1. **轻量铺路（仅当选项空间都不清时；用户输入已列候选①②… 或已有大致方向 → 直接跳到 2，不铺路）**：快速列 2-4 个**机制/取舍真差异**的方向（各一句价值 + 代价 + 适用 + **一个最大失败点/第一面墙**——让差异和风险一眼可见），让用户挑一个起点分支。方向集**须至少含一条"改上游/让问题不发生"候选**（全数同层实现变体=铺路失败）；铺不出=升 `/cw-think` 信号。
1.5 **两个非强制 offer（命中才提，不加仪式）**：① Human 在**陌生域**（不知道该问什么/不知道"好"长什么样）→ offer **盲区扫描**：先把该域的关键维度/常见坑/"好"的样子教给 Human（把"压根没考虑过"变成"知道自己没想清"），再回到逼问；② 方向涉**视觉/UX**（"看到才知道要不要"）→ offer **原型分叉**：2-4 个风格真差异的 HTML/示意稿先看再拍——这类未知在原型期发现便宜、实现期发现贵。
2. **顺着方向走"决策树"，逐个拍定**（进展的单位是"拍定一个决策"，不是"生成一个选项"）：
   - **默认一次只问一个，等回答再问下一个**（一次抛一堆=让人懵）。**三种显式例外可切批量**：Human 明示要批量 / 输入本身是多问题清单 / 收尾低风险叶子决策打包 react——批量时全部立场一次给齐 + 不确定点集中列尾，**决策仍逐个 resolve、不代拍**。
   - **按依赖排序**：先问"别的决策依赖它"的那个（A 依赖 B 就先定 B，别先定 A 再发现和 B 冲突）。
   - **每个问题带你的真立场推荐 + 为什么**——给明确倾向让用户 **react（同意/改）**，**别和稀泥**（"都行看你"=没尽到脑暴责任）；也别面对空白题让用户从头想。
   - **能从代码/现状/外部研究查到的，自己去查，别问用户**（省注意力；外部 scouting 如选型/竞品/资料标 ○ 直到实测——重型量化对比交 `/cw-experiment`）。下"不存在/需新建"结论前**穷尽同义词族（中英+缩写）+ 目标目录 ls + 先查已有共享库**；每个待拍决策先自答"**现状是否已满足**"（满足 = 报告事实，不上决策台）。
3. **逼到共识即停**：方向 + 影响"能不能建/怎么建"的**关键决策都拍定**（每个说得出"定它因为…、否掉别的因为…"），够交下游就停。**"决策都 resolved"才算完，不是"列了 N 个选项"算完。**

## 产出
拍定的**方向** + **已解决的关键决策清单**（每条：决策点 ｜ 定了什么 ｜ 为什么 ｜ 否掉了什么）。默认不落盘，结晶时 **offer 轻量决策记录**（决策清单 + **新术语表 Glossary**：本轮新词/定义，防跨 session 或多人交接时歧义——只 offer 不默认落）。**触碰既拍定项守卫**：本轮改动若触碰早前已拍定的决策 → 单列"触碰既拍定项清单（改了哪条/为什么）"交 Human 显式确认，**禁默认覆盖**；后续派工 prompt 附既拍定决策清单为不变量（既拍定项来源 = 本轮对话 + 可见决策记录/handoff；无历史可查 → 显式标"仅覆盖当前上下文"，不声称已扫历史）。**信念级深碰的资产不丢**：本轮实际发生信念修正（原以为→现在）/ 动了领域模型 → 结晶产物按 think 落盘规范**落** `docs/topics/`（登记 TOPIC-BOARD），哪怕没升 `/cw-think`——此为"默认不出文件"的显式例外（深碰资产必须跨 session 可取回）。

## DONE-WHEN
- 方向 + 关键决策都拍定（每个有 resolved 答案 + 理由）；下游/新 agent 拿到能直接建，**无需再回头问这些决策**。
- No-human/超时降级时：以"已拍定项 + [待人拍] 清单"的中间态收束（**不算 DONE**），Human 确认后闭合；未决项及其依赖不得开工。

## 去向
- 方向定、表达待打磨 → `/cw-clarify <scope>` → `/cw-spec`(新)/`/cw-change`(旧)。
- 方向定、够清晰 → 直接 `/cw-spec`/`/cw-change`。
- 选完发现**底层信念存疑** → 升 `/cw-think <scope>`。

## 纪律
- **不强制深挖第一性**（需要时可轻量碰一下帮拍板，但深碰不是它的重心；真要系统深碰核心信念 → `/cw-think`）；不收敛到单一可验诉求/不补验收细节（那是 clarify）；不建规格/不写码。
- **默认不出文件**（保持轻）；结晶时 **offer**："要把选中方向记下来喂下游吗？"，而非自动落盘。
- 方案"伪差异"(同思路换词) = 失败；轻量铺路时必须机制/取舍真差异。
- **逼问失败模式**：一次问一堆(让人懵、非例外情形) / 只丢选项不收敛(没拍定决策就算完) / 问能自己查代码的事 / 决策不按依赖排序(先定 A 后撞上没考虑的 B) / 问空白题不给推荐答案(让用户从头想) / **代拍决策**(事实自己查，**决策必须停下等 Human 拍**——agent 自问自答替人拍板=破坏共创) / **把技术开关空间当决策空间摆给 Human**(决策用业务场景语言，flag 退幕后) / **承重判断点压缩成黑话**(Human 读不懂=没交付)。
- **可选穷尽模式**：Human 明说 stress-test/exhaustive → 关键决策拍定后追加一轮遗漏扫描（风险/依赖/反例/不可逆点，各只问命中最高一项）。默认不开（保持轻）。
- **No-human 降级**（同 clarify）：无真人在场（unattended/pipeline）→ **不拍定**（也不出"已拍(pending)"式措辞），产出"决策点 + 推荐 + [假设]"待人确认清单，不阻塞；**不授权开建实现**；恢复有人时再确认。
- **问题组件超时 ≠ 拍定**：AskUserQuestion 超时（用户可能只是慢，不等于离场/同意）一律标 [待人拍]；**结构性决策**（schema/信息架构/模板/命名体系）超时不得开工实现，只推进不依赖它的批次；Human 回来先复述待拍清单让其 react。
